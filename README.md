# kostya

Монорепозиторий экосистемы Константина: три связанных Telegram-бота на общем каркасе (Python / aiogram 3 / PostgreSQL). Вместе они закрывают воронку **бесплатная аудитория → платный клуб → контент и знания эксперта**.

| Каталог | Продукт | Роль |
|---------|---------|------|
| [`club/`](club/) | Клубный бот («Любящие Бога» / Talk_God) | Платный клуб: подписка, доступ в группу, LLM-помощник, рассылки, админка |
| [`biblia/`](biblia/) | БиблияБот | Бесплатный бот по Писанию: диалог, донаты, цитаты, челлендж, молитва |
| [`avatar_kostya/`](avatar_kostya/) | Аватар Константина | Продюсер контента + RAG: индексация, Телемост, шортсы, рассылка в club/biblia |

```text
biblia (аудитория)  ──рефералы / кнопки──►  club (подписка)
                                              ▲
avatar_kostya (RAG + эфиры + шортсы) ─────────┘
        │
        └── #club / #biblia → mailing_campaigns в БД club / biblia
```

---

## Как устроена экосистема

1. **БиблияБот** приводит людей к Писанию и мягко знакомит с клубом.
2. **Клубный бот** принимает оплату, выдаёт доступ в закрытое сообщество и ведёт участника LLM-агентом.
3. **Аватар** наполняет базу знаний (RAG), обрабатывает эфиры Телемоста, нарезает шортсы/голосовые и ставит рассылки в очереди club/biblia.

Общий каркас ботов: `TelegramBotApp`, фичи в `bot/features/`, storage на **asyncpg**, платежи **YooKassa / BZB**, планировщик **APScheduler**.

## 1. `club/` — клубный бот

### Назначение

Telegram-бот **платного закрытого клуба**: онбординг, оплата подписки, вход в группу, персональный LLM-помощник в личке и операционная админка команды.

### Функциональность

**Участники**

- Старт и онбординг (профиль, рефералы, подарочная активация)
- LLM-диалог в личке (sales-агент для лидов / member-агент для участников)
- Оплата и подписка (`/payment`, `/subs`) — тарифы, карта, подарки
- Доступ в закрытую группу (`/club`) — одноразовые инвайты, контроль лицензии
- Поддержка и отзывы (`/support`, `/feedback`)
- Партнёрская программа (`/affiliate`)
- Раздел «Польза», юр. согласия (оферта / ПДн)
- Опционально: доска желаний, «Ангельский взнос», дожимы и напоминания о продлении

**Админы**

- Служебная супергруппа с топиками (оплаты, поддержка, диалоги)
- Reply в топиках → ответ пользователю в ЛС
- Отчёты, churn, воронки, мастер рассылок (`/new_mailing`)
- Сквозная аналитика Библия→Клуб (`/biblia_club`)
- Управление админами, промо, расписанием

### Стек

| Слой | Технологии |
|------|------------|
| Язык | Python 3 |
| Telegram | aiogram ≥ 3.0 (FSM, polling) |
| БД | PostgreSQL + asyncpg |
| LLM | OpenAI (диалоги, embeddings); DeepSeek — аналитика / churn (опц.) |
| RAG | ChromaDB (чтение индекса от аватара), tiktoken; опц. FastAPI read-API |
| Платежи | YooKassa, BZB |
| Прочее | httpx/aiohttp, APScheduler, ffmpeg, docx/PDF |
| Деплой | `scripts/deploy_prod.sh` → prod + supervisor |

### Связи

- Читает RAG из индекса **avatar_kostya** (сам не индексирует)
- Read-only к БД **biblia** для отчётов кампаний
- Принимает кампании рассылок, которые ставит аватар через `#club`

## 2. `biblia/` — БиблияБот

### Назначение

Бесплатный Telegram-бот **«БиблияБот»**: человек описывает ситуацию своими словами — бот подбирает отклик из Нового Завета и отвечает тепло, без нравоучений. Точка входа в экосистему клуба.

### Функциональность

- Диалог по Писанию (DeepSeek + история сообщений)
- Медиа → текст (голос, фото, видео, документы)
- FAQ и частые вопросы
- Донаты (`/payment`, `/donat`) — разовые и recurring
- Поддержка (`/support`)
- Реферальная программа (`/affiliate`, `/start ref_…`)
- Рассылки: воркер кампаний из БД; ежедневные цитаты и благословения
- Личная молитва (`/prayer`) — анкета → LLM → голос (Yandex SpeechKit)
- Челлендж чтения Писания (`/challenge`)
- Ежедневный админ-отчёт
- Кнопки перехода в клубный бот

### Стек

| Слой | Технологии |
|------|------------|
| Язык | Python 3 |
| Telegram | aiogram ≥ 3.0 |
| БД | PostgreSQL + asyncpg |
| LLM | DeepSeek (основной чат); OpenAI (вспомогательные задачи) |
| Платежи | YooKassa, BZB |
| TTS | Yandex SpeechKit |
| Планировщик | APScheduler |
| RAG | нет (контекст — история в Postgres) |
| Деплой | `scripts/deploy_prod.sh` → prod + supervisor |

### Связи

- Общий каркас и паттерны с **club**
- В UI — ссылки/кнопки на клубный бот
- Принимает кампании рассылок от аватара через `#biblia`

## 3. `avatar_kostya/` — аватар Константина

### Назначение

Telegram-бот — **цифровой аватар / контент-продюсер Константина**: идеи постов и роликов в личке на базе RAG; автоматическая загрузка знаний из групп, Яндекс.Диска и Телемоста; нарезка шортсов и голосовых; постановка рассылок в club и biblia.

Запускается из этого каталога (supervisor), без отдельного prod-копирования как у club/biblia.

### Функциональность

**Диалог и знания**

- Личный диалог (DeepSeek + RAG-контекст + «золотой фонд» удачных ответов)
- Индексация материалов из форум-групп Telegram (топики → метаданные → Chroma)
- Синхронизация Яндекс.Диска (WebDAV) в RAG
- Почта Телемоста (IMAP) → классификация → решение админа → индекс речи эксперта

**Эфиры и контент**

- Типы записей: Эфир, Молитва, Покаяние, Вопрос-ответ, Другое
- Полная аудиозапись в Telegram voice с LLM-названием и описанием
- Аудио-шортсы (мини-подкасты ~45–120 с) — только речь Константина; сначала целая мысль, потом нарезка
- Видео-шортсы (вертикаль + субтитры) — по флагу
- Молитвы: только RAG + полная запись (без нарезки шортсов)
- Ожидание письма со ссылками на видео/аудио перед скачиванием; fallback звука из видео, если `audio_only` урезан

**Рассылки в соседние боты**

- В топике шортсов: reply на медиа + `#club` / `#biblia`
- FSM: текст, CTA, аудитория, расписание
- Перезаливка media через токен целевого бота (`file_id` нельзя шарить между ботами)
- INSERT в `mailing_campaigns` / `mailing_audience` БД club или biblia

**Админ-команды (фрагмент)**

- `/telemost_status`, `/telemost_poll`, `/telemost_load`
- `/audio_cut [№]`, `/full_voice №`, текстом: `нарезать шортцы …`, `выложить полную запись встречи …`
- `/ydisk_status`, `/ydisk_sync`, `/rag_backfill`, `/rag_clear`, …

### Стек

| Слой | Технологии |
|------|------------|
| Язык | Python 3 |
| Telegram | aiogram ≥ 3.0 |
| БД | PostgreSQL + asyncpg |
| RAG | ChromaDB, OpenAI embeddings (`text-embedding-3-small`), tiktoken |
| LLM | DeepSeek (диалог); OpenAI (эмбеддинги, теги, моменты, классификация, подписи) |
| Медиа | ffmpeg, yt-dlp, Whisper (транскрипция) |
| Облако | WebDAV Яндекс.Диска, IMAP Яндекс.Почты |
| Платежи | YooKassa / BZB (донаты — наследие каркаса) |
| Запуск | supervisor из `dev/kostya/avatar_kostya` |

### Ключевые каталоги

| Путь | Роль |
|------|------|
| `rag/` | Chroma: чанки, индекс, retriever, golden store |
| `telemost_mail/` | IMAP, классификация, transcript, WebDAV/yt-dlp |
| `telemost_audio/` | Аудио-клипы, full voice, kinds, captions |
| `telemost_shorts/` | Видео-шортсы |
| `yandex_disk/` | Синк Диска → RAG |
| `bot/features/` | Фичи Telegram |
| `bot/services/shorts_mail/` | Кампании в БД club/biblia + reupload |

### Связи

- **Пишет** индекс RAG, который **читает** club
- **Ставит** рассылки в БД club и biblia; отправку делают их mailing-воркеры

## Быстрый старт (dev)

Каждый подпроект — свой `.env` и виртуальное окружение:

```bash
cd club      # или biblia / avatar_kostya
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
# заполнить .env
python main.py
```

Минимально нужны: токен бота, `DB_*`, ключи LLM (OpenAI / DeepSeek — по проекту). Детали — в `config.py` и `.env.example` (если есть) каждого каталога.
